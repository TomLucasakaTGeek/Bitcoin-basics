# Day 14: Lightning Addresses & Zaps

**Lightning Address:** How the industry turned complex invoices into human-readable identifiers like satoshi@getalby.com.

**The Metadata Layer:** How Lightning Addresses use the LUD-16 standard to translate an "email" into an LNURL-Pay request.

**Zapping:** The cultural phenomenon of sending micro-tips (as small as 1 sat) on social media platforms as a form of "super-like."

**Keysend:** Sending a payment to a node ID without requiring the recipient to generate an invoice first (Spontaneous Payments).

**Global Interoperability:** How a user on one app (e.g., Damus) can zap a user on another app (e.g., Primal) instantly across the globe.
